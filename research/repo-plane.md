# Планирование проекта

## Основные задачи

В данном проекте планируется реализовать прототип игры Mafia, в котором ai-агенты на основе slm (small language model) выступают в качестве акторов игры, а игрок (человек) в качестве ведущего, принимающего решения.

[Источник информации об игре mafia на wikipedia](https://ru.wikipedia.org/wiki/%D0%9C%D0%B0%D1%84%D0%B8%D1%8F_(%D0%B8%D0%B3%D1%80%D0%B0))

## Правила игры

Мы будем реализовывать собьственные правила игры по мотивам Mafia:

- игре принимают участие 10 ai-агентов, для которых заданы системные промпты, определяющие их характер и манера общения.
- системные промпты должны ограничить агентов так, чтобы те не давали ответы на вопросы, не относящиеся к игре. Это необходимо для того, чтобы диалог между агентами не уходил в сторону.
- в начале игры каждый агент получает роль: 7 агентов становятся горожанами, а 3 агента - мафией.
- игрок не знает кто мафиози, а кто горожанин.
- игра делится на раунды, а каждый раунд на фазы - ночь, ночное голосование, день, дневное голосование и выбор ведущего.
- агенты обмениваются сообщениями ночью и днем, а затем голосуют
- кто является мафией наверняка знают только мафиози.
- ночью сообщениями может обмениваться только мафия. Горожане сообщения мафии не слышат.
- во время обмена сообщениями мафия должна договориться кого из горожан им убить.
- после ночного обмена сообщениями каждый мафиози объявляет имя жертвы. Если все голоса отданы за одного агента, этот агент выбывает из игры.
- днем горожане и мафиози обсуждают кто из агентов может быть мафией. Агенты могут врать, заявлять что мафия это они и так далее - главное, что никто наверняка не знает правды.
- после дневного обсуждения наступает тайное голосование - агенты объявляют кого они хотят казнить.
- результат дневного голосования публикуется и становится известен агентам и игроку.
- игрок может согласиться с результатом голосвания, а может не согласиться.
- если игрок согласился и голосованием определен победитель, набравший абсолютное большинство голосов - этот агент выбывает из игры.
- если в голосвании несколько победителей, игрок может выбрать кого-то из них.
- если игрок не согласен, то ничего не происходит - никто не выбывает из игры.
- в игре побеждает мафия, если выбывают все горожане.
- в игре побеждают горожане, если из игры выбывают вся мафия.

## Программная реализация

Игра должна быть реализована следующим образом:

- все сущности должны быть реализованы в docker-контейнерах
- весь стек контейнеров должен запускаться с помощью docker-compose
- весь процессинг генерации диалогов должна осуществлять модель mistral-6b, запускаемая в отдельном контейнере с помощью библиотеки ollama
- другим контейнерам в docker-сети контейнер с моделью должен предоставить api в формате mcp
- агенты должны подниматься каждый в собственном docker-контейнере и хранить собственное состояние (т.е. знать свой характер, манеру общения, текущую роль и помнить сообщения, которые они получали)
- обмен сообщениями между агентами должен осуществляться по протоколу, который обеспечит получение этих сообщений всеми участниками игры в зависимости от их ролей (горожанин/мафиози), включая игрока. Предпочтительна очередь типа RabbitMQ или аналог pub/sub для неограничегнного числа подписчиков.
- агенты должны обращаться в контейнер с llm-моделью, очищать ее контекст, загружать свой системный промпт и другие текущие тексты, необходимые для формирования сообщений, затем отправлять сформированное сообщение другим агентам и игроку. Таким образом будет осуществляться общение.
- должен быть предложен разумный вариант упорядоченности общения, чтобы агенты не перебивали друг друга и не перегружали модель и в тоже время имели возможность высказаться. Предпочтителен асинхронный вариант публикации сообщений.
- время, отведенное на общение агентов внутри фаз ночь и день должно быть ограничено.
- игроку должна быть предоставлена админ-панель, в которой он должен видеть сообщения агентов, текущий раунд и фазу игры, а так-же число не выбывших агентов. В панели должен быть реализован интерфейс одобрения результатов голосвания и принятия решения.
- предпочтительный язык реализации для всех контейнеров - python. Прототип админ-панели можно написать на фреймворке streamlit
